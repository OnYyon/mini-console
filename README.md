## Описание
Приложение имметирующее shell linux.
Запуск:
```sh
uv run python3 -m src.main <command>
```
или через `Makefile`
```sh
make run test
```
А дальще просто менять команды в Makefile

---

## Реализования команды
   - ``cd``
   - ``ls``
   - ``pwd``
   - ``mv``
   - ``cp``
   - ``rm``
   - ``zip``
   - ``unzip``
   - ``tar``
   - ``untar``
   - ``grep``
   - ``history``
   - ``cat``
   - ``undo``

## Некторые методы применные для реализации проекта
   - Создано окружение в ``.env,`` где храниться текущий путь(после всех ``cd``) и текущий счетчик выполения команд
   - Созданны декараторы для логирование в человесокм формате и в машинном(json) для выполенения ``undo``.

---

## Структура проекта
- ``./src/commands`` все файлы обработчкики команд для typer
- ``./src/utils`` файлы утилиты для использования в нескольких файлов
- ``./src/logger`` файлы для логирования
- ``./.env`` переменное окржение для хранения ``текущего пути`` и счетчика для команды ``history``

## Самокртика
Вообще было лучше разделить хендлеры и само исполнения команд. Тогда было бы проще делать тесты и пробрасывать своих ``менеджерей`` для более удбной обработки ошибок или управления переменным окружением, так же для более чистой архитиктуре.
``typer`` - достаточно высокоуровневая библиотека в некоторыех моментах для большего контроля я бы пока использовать argparse, но это не точно.
Как подчинить себе mypy? Прокидовал в конекст тайпера свои параментры для удобного логирования в json, а он ругаеться, что конекст это класс(как я понял) и у него нету такого атрибиту. Пробывал и добавлять атрибуты и проверять их наличия, а потом создавть. Пришлось делать ``# type: ignore``
